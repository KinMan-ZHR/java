# java
java 设计模式
1. 单例模式
2. 工厂模式
   ```text
   定义一个订单接口，并实现多个具体订单。
   1. 订单接口：定义订单的属性和行为。
   2. 具体订单：实现订单接口，定义订单的属性和行为。
   3. 订单工厂：定义创建订单的工厂方法，根据传入的参数创建不同的订单对象。
   4. 客户端：通过工厂方法创建订单对象，并调用订单对象的相关方法。
   ```
3. 构建模式 
   ```text
   当需要构建的对象具有复杂的内部结构，且构建过程需要多个步骤时，可以使用构建者模式将构建过程拆分成多个独立的步骤。
   当需要创建不同表示形式的对象时，可以使用构建者模式。通过使用不同的具体构建者，可以构建出不同属性和行为的对象。
   当需要灵活地组合和构建对象时，可以使用构建者模式。通过定义不同的具体构建者和指导者，可以按需组合构建对象。
   产品（Product）：表示被构建的复杂对象。通常包含多个属性和方法。
   抽象构建者（Builder）：定义了构建产品对象的抽象接口。通常包含设置产品属性的方法和获取构建结果的方法。
   具体构建者（Concrete Builder）：实现了抽象构建者接口，具体实现了构建产品对象的方法。它负责实现构建过程中的每一步细节。
   指导者（Director）：负责调用具体构建者来构建产品对象。它不直接与产品交互，而是通过抽象构建者来构建产品。
   1. 产品：定义产品的属性和行为。
   2. 私有化构造函数：阻止外部直接创建产品对象，只能通过构建者创建。
   3. 定义静态内部类（构建者），定义构建产品的步骤。
   4. 定义指导者，定义构建产品的流程，通过构建者创建产品对象。
   5. 客户端：通过指导者创建产品对象，并调用产品的相关方法。
   ```   
4. 原型模式
   ```text
    当需要复制一个对象时，可以使用原型模式。原型模式通过复制原型对象来创建新的对象，从而避免了直接创建对象时的大量重复工作。
    1. 原型接口：定义复制对象的方法。
    2. 具体原型：实现原型接口，定义复制对象的方法。
    3. 客户端：通过原型接口复制对象，并调用对象的相关方法。
   ```
5. 代理模式
   ```text
   当需要控制对对象的访问时，可以使用代理模式。代理模式通过创建一个代理对象，代理对象代理目标对象，从而控制对目标对象的访问。
   类似AOP（Aspect Oriented Programming）编程。用于拦截对目标对象的访问，并进行预处理、后处理等操作。
   1. 目标接口：定义目标对象的接口。
   2. 目标对象：实现目标接口，定义目标对象的行为。
   3. 代理对象：实现目标接口，代理目标对象，控制对目标对象的访问。
   4. 客户端：通过代理对象访问目标对象，并调用目标对象的相关方法。
   ```
6. 桥接模式
   ```text
   当需要将抽象与实现分离时，可以使用桥接模式。桥接模式通过将抽象与实现分离，将抽象与实现解耦，从而实现对抽象和实现进行灵活的扩展。
   将继承的问题转化为组合问题。
   1. 抽象接口：定义抽象对象的接口。
   2. 实现接口：定义实现对象的接口。
   ```
7. 装饰器模式
    ```text
       当需要动态地给一个对象添加一些额外的职责时，可以使用装饰器模式。装饰器模式通过将额外的职责包装在装饰器对象中，将这些职责与核心对象解耦，从而实现对核心对象的扩展。
       1. 抽象组件：定义组件的接口。
       2. 具体组件：实现抽象组件接口，定义组件的行为。
       3. 抽象装饰：定义装饰的接口，继承抽象组件接口。
       4. 具体装饰：实现抽象装饰接口，装饰具体组件对象，扩展具体组件对象的功能。
       5. 客户端：通过装饰器对象装饰具体组件对象，并调用对象的相关方法。
    ```
8. 适配器模式
   ```text
   当需要将一个类的接口转换为另一个类的接口时，可以使用适配器模式。适配器模式通过创建一个适配器对象，将一个类的接口适配到另一个类的接口，从而实现对两个接口的兼容。
   1. 源接口：定义源对象的接口。
   2. 目标接口：定义目标对象的接口。
   3. 适配器：实现目标接口，通过源对象实现目标接口的功能。
   4. 客户端：通过适配器对象调用目标对象的相关方法。
   ```
9. 外观模式
   ```text
   组合模式由以下几个核心组件构成：
   组件（Component）：定义了组合对象和叶子对象的共同行为。它可以是接口或抽象类，提供了统一的接口用于访问组合对象和叶子对象。
   叶子对象（Leaf）：表示组合对象中的叶子节点，它没有子节点。它实现了组件接口，并定义了自己的行为。
   组合对象（Composite）：表示组合对象中的非叶子节点，它包含了子节点。它实现了组件接口，并可以执行一系列操作，例如添加子节点、删除子节点和遍历子节点等。
   使用场景
   当存在部分-整体的层次关系，并且希望统一处理单个对象和组合对象时，可以使用组合模式。它提供了一致的方式访问对象，并简化了代码的设计和使用。
   当需要对对象进行递归组合和遍历操作时，组合模式非常有用。它可以让我们以统一的方式处理组合对象和叶子对象，无需进行类型判断。
   当希望将多个对象组织成树状结构，并对整个结构进行操作时，可以使用组合模式。它提供了方便的方式来管理和操作对象的层次结构。

   ```
10. 享元模式
   ```text
   Flyweight（享元）：定义共享对象的接口，包含对共享状态的操作方法。
   ConcreteFlyweight（具体享元）：实现享元接口，并实现共享状态的方法。
   FlyweightFactory（享元工厂）：负责创建和管理享元对象，提供对共享对象的访问和复用。
   ```
